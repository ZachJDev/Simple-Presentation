Case 3

  • Tools:

    • APEX

  • Issue:

    • Users reported a bug where, if they chose to 'cancel' a sub-page, they would lose all the work completed on the main page

  • Resolution:

    • Upon inspection, discovered that the feedback 'page' was actually just a number of page elements being hidden and show.
    • However, the page was refreshed when the 'cancel' button was used.
    • From there, I was able to simply reverse the steps taken to show the page to instead hide it.
Case #2 Case #4

Case 4

  • Tools:

    • DevTools, CSS, APEX

  • Issue:

    • An application user reported than when attempting to print a report, the report would sometimes be pushed to a second page, leaving large gap on the first page below the page header.

  • Resolution:

    • Inspecting the CSS template in APEX, I found that there was a directive to avoid page breaking in the middle of a ‹td› element.
    • Using the browser Developer Tools, I realized that APEX uses tables for it's layout, and the entire report was embedded in a single ‹td› element.
    • I rectified this by assigning a class to the ‹td›s of the report, then rewriting the template to avoid breaking on that class.

Docker/Nginx Demo

  1. Objective:

    1. Demonstrate Nginx's load-balancing capabilities.

  2. Methodology:

    1. Create a Node.js/Express server that receives a request and waits for 3 seconds before responding to it.
    2. Spin up 3 instances of the server using Docker to manage their execution.
    3. launch an Nginx load-balancing server and route requests to the different Node.js servers sequentially.

  3. Expected Outcome:

    1. If we submit 3 simultaneous curl requests to a single Node server, we should expect each to be processed individually, receiving each after 3 seconds, with a total time of 9 seconds for all requests
    2. If we submit 3 simultaneous requests to the load balancing server, we should expect a single batch of 3 responses after 3 seconds.
